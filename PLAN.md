# План развития проекта: маникюр-мастер

---

## Фаза 1 — MVP: что-то живое как можно быстрее

Цель: мастер получает первого клиента через сайт.
Готово, когда: сайт открывается по ссылке, мастер его одобрила, кнопка «Записаться» работает.

- [ ] Согласовать дизайн с мастером (цвета, стиль, фото)
- [ ] Собрать данные: услуги, цены, длительность каждой, рабочие часы
- [ ] Лендинг (имя, услуги с ценами, фото, кнопка «Записаться»)
- [ ] Кнопка ведёт в Telegram-бота или в чат напрямую
- [ ] Выбрать и настроить хостинг
- [ ] Опубликовать сайт в интернете
- [ ] Базовый SEO: метатеги, заголовок страницы, описание

---

## Фаза 2 — Онлайн-запись

Цель: клиент записывается без переписки, мастер видит расписание.
Готово, когда: реальный клиент успешно записался через форму, запись отобразилась в панели мастера.

- [ ] Выбрать архитектуру: бот-первый (Telegram — главный вход) или web-первый (сайт — главный вход, бот только уведомляет)
- [ ] Решить, где хранятся данные (Google Sheets, база данных или только бот)
- [ ] Решить, как синхронизируются сайт и бот (одна запись — один источник данных)
- [ ] Карточка клиента (имя, телефон, история визитов, предпочтения)
- [ ] Форма выбора услуги, даты и времени
- [ ] Учёт длительности каждой услуги (разные слоты для разных услуг)
- [ ] Буферное время между клиентами
- [ ] Верификация клиента через Telegram-бота
- [ ] Страница подтверждения записи
- [ ] Защита от двойного бронирования одного слота
- [ ] Панель мастера (просмотр записей, блокировка времени)
- [ ] Повторяющиеся блокировки (выходные, обед каждый день)
- [ ] Авторизация для панели мастера (пароль или вход по коду)

---

## Фаза 3 — Telegram-бот

Цель: клиент получает напоминания, мастер — уведомления.
Готово, когда: тестовый клиент получил напоминание, подтвердил визит, мастер увидела уведомление.

- [ ] Напоминание клиенту за день до визита
- [ ] Подтверждение / перенос / отмена визита
- [ ] Уведомление мастеру о новой записи
- [ ] Уведомление мастеру об отмене или переносе
- [ ] Уведомление клиенту из листа ожидания при освобождении слота
- [ ] Запрос отзыва после визита

## Технические ограничения (решить до фазы 2)

Лендинг и визитка — статические HTML-страницы, они работают по правилам проекта.
Онлайн-запись — уже другой класс задач: нужен бэкенд, хранилище данных, API бота.
Для фазы 2 правила придётся расширить.

**Рекомендуемый стек** (оптимально для учебного проекта):
Telegram-бот на Python + статический HTML-лендинг + Google Sheets как база данных.
Это даёт рабочий результат за 1–4 недели без дорогой инфраструктуры.

- [ ] Утвердить стек: Telegram-бот (Python) + Google Sheets — или выбрать другой
- [ ] Решить, нужен ли отдельный сервер или хватит бесплатного хостинга (Railway, Render)
- [ ] Зафиксировать обновлённые правила проекта для динамической части

---

## Контент (подготовить до начала вёрстки)

Без этого разработка встанет — не из чего собирать страницы.

- [ ] Мастер пишет тексты о себе (кто она, опыт, подход к работе)
- [ ] Мастер предоставляет фото работ (минимум 6-10 штук, хорошего качества)
- [ ] Мастер составляет прайс: название услуги, цена, длительность
- [ ] Согласовать имя и фото для лендинга и визитки

---

## Бюджет (обсудить с мастером)

- [ ] Хостинг: от 0 ₽ (GitHub Pages/Netlify) до ~300 ₽/мес (VPS)
- [ ] Домен: ~700–1000 ₽/год (опционально, можно начать без него)
- [ ] Бэкенд-сервис (если нужен): часть сервисов бесплатны на старте
- [ ] Договориться, кто оплачивает инфраструктуру
