# Проект: онлайн-запись для маникюрного мастера

## Что это за проект

Кастомный цифровой продукт для мастера маникюра: клиенты записываются онлайн сами, получают напоминания и могут подтвердить или перенести визит без переписки. Мастер видит всё расписание в одном месте, управляет временем и показывает свои работы. Проект включает лендинг, систему онлайн-записи, Telegram-бота и галерею работ.

> **Почему не DIKIDI или EasyWeek?**
> На рынке есть бесплатные готовые решения (DIKIDI, EasyWeek). Мы строим кастомное — осознанно: это учебный проект по вайб-кодингу, и конечный продукт будет заточен под конкретного мастера без лишнего функционала и без чужого брендинга.

---

## Для кого

- **Клиенты мастера** — записываются самостоятельно, выбирают удобное время, получают напоминания
- **Мастер маникюра** — ведёт запись без хаоса в мессенджерах, управляет расписанием, показывает свои работы

## Архитектура (открытые вопросы)

Эти вопросы надо решить до начала разработки онлайн-записи:

- **Где хранятся данные?** Google Sheets, база данных или только в боте
- **Как синхронизируются сайт и бот?** Запись через сайт должна сразу закрывать слот в боте — и наоборот
- **Резервный канал:** если сайт или бот недоступен, клиент должен знать запасной способ связи

## Конкурентное преимущество

Слабые места конкурентов — это наш шанс:

| Проблема у конкурентов | Наше решение |
|---|---|
| Сложный интерфейс (YCLIENTS) | Минимум шагов, только нужное |
| Автоуведомления только за деньги (DIKIDI) | Напоминания работают сразу |
| Регистрация перед записью | Записываться без аккаунта |
| Плохая сетка слотов на мобильном | Адаптив от 320px — приоритет |
| Дорого для одного мастера | Создано под одного мастера |

## UX-принципы

Правила, которые нельзя нарушать при проектировании:

- **Без регистрации** — клиент записывается без создания аккаунта
- **Минимум полей** — только имя и телефон (или Telegram)
- **Мгновенное подтверждение** — клиент сразу видит, что запись прошла
- **Мобильный — приоритет** — большинство клиентов придут с телефона
- **Перенос прямо из уведомления** — не нужно заходить на сайт отдельно

## Безопасность

- Панель мастера закрыта паролем или кодом входа
- Форма записи защищена от двойного бронирования одного слота
- Верификация клиента через Telegram (чтобы нельзя было оставить фейковый номер)

---

## Правила и политики

- Политика отмены: за сколько часов клиент может отменить без последствий
- Предоплата или предупреждение о no-show — решается с мастером
- Политика конфиденциальности обязательна (152-ФЗ, сбор имён и телефонов)

## Стиль дизайна

Финальный стиль согласовывается с мастером до начала вёрстки.

Базовые правила (из CLAUDE.md):
- Тёмный фон, светлый текст, оранжевый акцент
- Адаптивная вёрстка от 320px
- Анимации только на CSS и нативном JS
- Всё в одном HTML-файле, без внешних библиотек

> Важно: маникюрный бизнес часто использует нежную, "чистую" эстетику.
> Стоит показать мастеру несколько вариантов и выбрать вместе.

## Инфраструктура

- **Хостинг:** выбирается совместно с мастером (GitHub Pages, Netlify или VPS)
- **Домен:** при необходимости регистрируется отдельно (~700–1000 ₽/год)
- **Данные:** хранилище выбирается на этапе разработки онлайн-записи

---

## Технические ограничения

Фаза 1 (лендинг, визитка) — статические HTML-страницы, работают без бэкенда.
Фаза 2+ (онлайн-запись, бот) — требует бэкенда и хранилища данных. Правила проекта для этих частей будут расширены отдельно.

Варианты для фазы 2:
- Сервис без кода: n8n, Make, Google Apps Script
- Полноценный бэкенд: Python или Node.js

- Имя и фото для лендинга и визитки
