## 1. Цель сайта и целевая аудитория

### Цель

Конвертировать посетителя из Instagram или поиска в запись через Telegram.
Не продавать маникюр как услугу — продавать **доверие к конкретному мастеру**.

Одна страница отвечает на пять вопросов клиента:
1. Мне нравится её стиль работ?
2. Сколько это стоит?
3. Где она принимает — далеко ли мне ехать?
4. Ей можно доверять? Что говорят другие?
5. Как записаться — это просто?

### Целевая аудитория

| Параметр | Описание |
|---|---|
| Пол / возраст | Женщины, 20–45 лет |
| География | Ростов-на-Дону |
| Устройство | 70–85% заходят с телефона |
| Источник | Instagram → ссылка в шапке профиля |
| Мотивация | Ищет «своего» мастера, устала от очередей или неудачных результатов |
| Страхи | Долго ехать, дорого, не нравится стиль, плохой мастер, сложно записаться |

## 4. Стиль дизайна

### Цветовая палитра

```css
--bg:          #070f0c   /* Фон страницы — почти чёрный с мятным оттенком */
--card:        #0c1914   /* Фон карточек и чередующихся секций */
--orange:      #16c47f   /* Акцент — мятный зелёный */
--orange-glow: #2de89a   /* Акцент светлее — ховеры, цифры */
--text:        #d5f0e8   /* Основной текст */
--muted:       #4a7a63   /* Второстепенный текст, подписи */
--border:      rgba(255, 255, 255, 0.06)  /* Границы карточек */
```

**Настроение:** свежесть, профессионализм, чистота. Выделяется на фоне конкурентов — 80% nail-сайтов используют розовый и бежевый.

**Дополнительный тёплый акцент** (рекомендация из research.md): персиковый `#E8A87C` для декоративных элементов — смягчает «технический» вид мятной палитры. Пока не внедрён.

### Шрифты

Сейчас используется системный стек: `-apple-system, BlinkMacSystemFont, 'Segoe UI', system-ui, sans-serif`
**Плюс:** грузится мгновенно, не требует внешних запросов.
**Минус:** нет выразительного serif-акцента в заголовках.

Рекомендованная пара (из research.md): **Playfair Display** (заголовки) + **Inter** (текст). Требует Google Fonts, что противоречит правилам проекта (нет внешних библиотек). Решение — согласовать с заказчиком отдельно.

### Типографика

| Элемент | Размер | Вес |
|---|---|---|
| Заголовок H1 | `clamp(2.2rem, 6vw, 4rem)` | 800 |
| Заголовок H2 | `clamp(1.75rem, 4vw, 2.75rem)` | 800 |
| Основной текст | 1rem (16px min) | 400 |
| Подзаголовки, мелкие | 0.875rem | 400 |
| Лейблы секций | 0.8rem, uppercase, letter-spacing | 700 |

### Анимации

| Эффект | Реализация | Где используется |
|---|---|---|
| Появление при скролле | `IntersectionObserver` + CSS opacity/translateY | Все секции |
| Анимация при загрузке | CSS `@keyframes fadeUp` с задержками | Hero |
| Shimmer на кнопке | CSS `::after` + `@keyframes btn-shimmer` | `.btn-primary` |
| Пульс точки «онлайн» | CSS `@keyframes pulse` | Hero badge |
| Частицы на фоне | JS + CSS `@keyframes rise` | Фон страницы |
| Hover-эффекты | CSS `transform: translateY(-4px)` | Карточки |
| SVG-волны | Inline SVG | Между секциями |

### Разделители секций

SVG-волны двух типов:
- **bg → card:** `rect fill="#070f0c"` + `path fill="#0c1914"` (волна вниз)
- **card → bg:** `rect fill="#0c1914"` + `path fill="#070f0c"` (волна вверх)

## 5. Технические требования

### Обязательные правила (CLAUDE.md)

- Один HTML-файл. CSS и JS — встроены через `<style>` и `<script>`.
- Никаких внешних библиотек: без Bootstrap, jQuery, React и т.п.
- Никаких отдельных `.css` и `.js` файлов.
- Адаптивность от 320px: `meta viewport`, flexbox/grid, `rem/%/vw`.

### Ключевые элементы UX

| Элемент | Требование |
|---|---|
| Кнопка «Записаться» | Видна на каждом экране (nav + hero + #how + sticky-mobile) |
| Sticky CTA | Только на мобильных (≤ 640px), появляется после hero |
| Минимальный размер кнопки | 44×44px (Apple/Google recommendation) |
| Минимальный шрифт | 16px на мобильном |
| Портфолио мобайл | CSS Scroll Snap, горизонтальный свайп |
| Производительность | Изображения сжаты, нет внешних запросов |

### Адаптивные брейкпоинты

| Ширина | Изменения |
|---|---|
| ≤ 900px | Сетки услуг и портфолио: 3 → 2 колонки |
| ≤ 860px | Блок «О мастере»: 2 колонки → 1 (фото над текстом) |
| ≤ 700px | Отзывы: 2 → 1 колонка |
| ≤ 640px | Sticky CTA включается. Волны меньше (52px → 32px) |
| ≤ 560px | Сетка услуг: 1 колонка |
| ≤ 500px | Портфолио: горизонтальный скролл. Преимущества: 2 → 1 |
| ≤ 480px | Hero: скрывается кнопка «Смотреть работы», убираются разделители |

## 9. ПЛАН РАЗРАБОТКИ: TELEGRAM MINI APP

**Что это:** каталог услуг мастера маникюра внутри Telegram.
Клиент открывает бота → нажимает кнопку → видит услуги с ценами → выбирает → нажимает «Записаться» → бот сообщает Ольге.

**Файл:** `olga-project/tma/index.html` (отдельный файл, не лендинг)
**Хостинг:** тот же Vercel, путь `/tma`
**Бот:** создать через @BotFather, подключить к `/tma` как WebApp

### Переходы между экранами (навигация)

```
[Главный]
    │  тап на категорию
    ▼
[Каталог] ──── BackButton ────▶ [Главный]
    │  тап на услугу
    ▼
[Карточка] ──── BackButton ───▶ [Каталог]
    │  MainButton «Записаться»
    ▼
[Подтверждение] ── BackButton ▶ [Карточка]
    │  MainButton «Отправить заявку»
    ▼
[Успех]
    │  MainButton «Готово»
    ▼
  закрыть TMA
```

Анимация переходов: `translateX(100% → 0)` при движении вперёд, `(0 → 100%)` при возврате. Длительность 250ms, `ease-out`.

### Что передаётся боту при отправке

`tg.sendData()` отправляет JSON-строку:

```json
{
  "service_name": "Маникюр с гель-лаком",
  "service_price": "1500",
  "service_duration": "90",
  "user_id": 123456789,
  "user_name": "Анна",
  "user_username": "anna_tg"
}
```

Бот получает это, формирует сообщение Ольге:
> 📩 Новая заявка!
> Услуга: Маникюр с гель-лаком · 1500 ₽ · 90 мин
> Клиент: Анна (@anna_tg)
> Ответь клиенту: @anna_tg

### Дизайн-система

**Цвета:** CSS-переменные Telegram — не хардкодить. Поверх добавить только акцентный:
```css
--accent: #16c47f;  /* тот же мятный, что на лендинге */
```

**Шрифт:** системный `-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif` — нативный вид внутри Telegram.

**Размеры:**

| Элемент | Размер |
|---|---|
| Карточки категорий | высота 56px, padding 16px |
| Карточки каталога | высота 72px, padding 12px 16px |
| Фото на карточке услуги | ширина 100%, соотношение 4:3 |
| Фильтры-чипы | высота 36px, padding 0 16px |
| Все кнопки | минимум 44×44px |
| Основной текст | 15px |
| Цена | 24–28px, жирный, `--accent` |
| Подписи | 13px, `var(--tg-theme-hint-color)` |

**Отступы:**

| Контекст | Значение |
|---|---|
| Горизонтальные поля | 16px |
| Между карточками | 8px |
| Между секциями | 24px |
| Отступ снизу (под MainButton) | 80px |

### Чего НЕ будет в первой версии

| Функция | Причина отсутствия | Когда добавить |
|---|---|---|
| Выбор даты и времени | Требует backend или Google Sheets API. Сложно без сервера. | Фаза 2 |
| Онлайн-оплата (Telegram Pay) | Нужна регистрация как юрлицо, подключение эквайринга | Фаза 3 |
| Профиль клиента / история записей | Нужна база данных | Фаза 2–3 |
| Напоминания за 24 часа | Нужен cron-job на сервере | Фаза 2 |
| Отмена записи через TMA | Нет БД, нет состояния записей | Фаза 2 |
| Фильтр по мастеру | Ольга одна — не нужен | Когда появится второй мастер |
| Тёмная / светлая тема — переключатель | TMA подхватывает тему Telegram автоматически | Не нужно |
| Авторизация / регистрация | Telegram даёт имя и ID — регистрация не нужна никогда | — |
| Сетка 2 колонки в каталоге | Вертикальный список удобнее читать цены | — |

### Стек и файлы

```
olga-project/
├── index.html          ← лендинг (готово)
└── tma/
    └── index.html      ← Telegram Mini App (план)
```

**Единственная внешняя зависимость:**
```html
<script src="https://telegram.org/js/telegram-web-app.js"></script>
```
Это официальный SDK Telegram — не библиотека, обязательный компонент платформы.

**Данные услуг:** JSON-объект прямо в скрипте, без внешнего API.
