## 3. UX И СЦЕНАРИИ

### Сценарий 1 — Клиент записывается впервые

Клиент видит ссылку в Instagram мастера. Открывает страницу: имя, фото работ, список услуг с ценами. Выбирает услугу → выбирает дату и время из сетки → вводит имя и телефон → подтверждает. Получает сообщение в Telegram: «Вы записаны на маникюр 5 марта в 14:00».

**Что удобно:** Минимум полей (имя + телефон), наглядная сетка слотов, мгновенное подтверждение.

**Что бесит:** Требование зарегистрироваться перед записью. Слишком много шагов. Сетка неудобна на мобильном. Нет подтверждения — непонятно, прошла ли запись.

---

### Сценарий 2 — Клиент переносит запись

Клиент получает напоминание за день до визита. Понимает, что не сможет прийти. Нажимает «Перенести» прямо в сообщении → видит доступные слоты → выбирает новое время. Мастер получает уведомление об изменении, освободившийся слот снова открывается для записи.

**Что удобно:** Кнопка переноса прямо в уведомлении. Мастер видит изменения сразу.

**Что бесит:** Отмена только через звонок или переписку. Нет возможности перенести — только отменить. Мастер узнаёт об отмене случайно, через несколько часов.

---

### Сценарий 3 — Мастер управляет расписанием

Мастер открывает приложение утром. Видит список на день: 4 клиента, первый в 10:00. Один попросил добавить дизайн — мастер вручную меняет продолжительность визита. Вечером ставит выходной на субботу — слоты закрываются автоматически. Просматривает записи на неделю вперёд.

**Что удобно:** Быстрый обзор дня одним экраном. Простое добавление выходного. Уведомление, если клиент отменил.

**Что бесит:** Нет вида на неделю целиком — только по дням. Нельзя принять запись вручную через приложение. Сложная настройка при нестандартном графике (через день, с 11 до 20).

## 5. МОНЕТИЗАЦИЯ

### Как зарабатывают конкуренты

**Freemium → Подписка** — самая распространённая модель (DIKIDI, YCLIENTS, Rubitime, EasyWeek). Базовый функционал бесплатно, платишь за автоуведомления, аналитику, несколько сотрудников. Работает: мастер привыкает бесплатно, потом платит за то, что реально нужно. Психологический порог — до 500–700 руб./мес.

**Платные уведомления** — SMS продаются отдельными пакетами поверх подписки (~3 руб./SMS, пакет 100 шт. за 200 руб.). Прозрачная и принятая модель.

**Комиссия с записи** — 5–15% с каждого завершённого визита. Работает только если платформа сама привлекает клиентов через каталог. Риск: мастера уходят в прямую переписку, чтобы не платить.

**Реклама в каталоге** — мастер платит за продвижение выше в поиске (DIKIDI Online, YPLACES). Работает при наличии трафика в каталоге.

**White-label** — готовая платформа под брендом сети. Только для крупных клиентов (10+ точек).

**Партнёрства с поставщиками** — реферальный процент с заказов материалов (гель-лаки, инструменты). Редко, но перспективно.

---

### Вывод для нашего проекта

Для одиночного мастера на старте реально работают две вещи:

1. **Freemium.** Мастер получает реальную ценность бесплатно (запись + расписание + напоминания). Платный тариф — когда видит результат.
2. **Платные SMS-пакеты.** Понятная модель — платишь за то, что отправляешь.

Комиссия и реклама в каталоге — только после набора собственного трафика.

## 9. UX И КОНВЕРСИЯ BEAUTY-ЛЕНДИНГОВ

### Топ-5 факторов доверия (что убеждает записаться)

1. **Реальные фото работ** — главный аргумент. Без них конверсия падает в 3–5 раз
2. **Лицо мастера** — фото живого человека резко повышает доверие
3. **Отзывы с именем и конкретикой** — «Хожу 2 года, беру дочку» > «Всё понравилось»
4. **Прозрачные цены** — скрытые цены = потеря 40–60% клиентов
5. **Понятная логистика** — где, как, когда; неопределённость = потеря клиента

### Типичные ошибки beauty-лендингов

- Нет конкретного предложения в первом экране (расплывчатый заголовок)
- Фото плохого качества или стоковые
- Кнопка записи только одна — в самом низу
- Нет мобильной версии (70–85% приходят с телефона)
- Отсутствие ответа «почему именно вы»
- Устаревшие отзывы (3–4 летней давности) или их отсутствие

### Лучшие CTA-практики

- Писать действием: «Записаться на маникюр» вместо «Оставить заявку»
- Микротекст под кнопкой: «Отвечаю в течение 1 часа · Запись без предоплаты»
- Одна главная кнопка на экран, не три разных
- Sticky-кнопка на мобильном — прикреплена к нижнему краю при скролле
- Никогда не писать «Отправить» на кнопке формы

### Мобильный UX — критично

- Минимальный размер кнопки: 44×44px (Apple/Google recommendation)
- Телефон кликабельный: `<a href="tel:+7...">` — обязательно
- Галерея — свайпом горизонтально, как в Instagram
- Минимальный размер шрифта: 16px
- Страница грузится до 3 сек — иначе теряется ~40% посетителей

### Социальные доказательства (по эффективности)

| Тип | Эффективность | Примечание |
|---|---|---|
| Рейтинг Google/Яндекс (независимый) | Очень высокая | Виджет со звёздами + число отзывов |
| Скриншоты переписки с клиентами | Высокая | Выглядит правдиво, не причёсано |
| Видеоотзывы (15–30 сек) | Высокая | Трудно подделать |
| Цифры с контекстом | Средняя | «800 клиентов за 5 лет» > «5 лет опыта» |
| Анонимные текстовые отзывы | Низкая | Выглядят как выдуманные |

| Гармоничная | DM Serif Display | DM Sans | Современная |
| Тёплая | Fraunces | Nunito Sans | Дружелюбная |
| Надёжная | Libre Baskerville | Lato | Консервативная |

**Важно:** Системный стек (`-apple-system, BlinkMacSystemFont`) грузится быстрее всего и хорошо работает на мобильном. Google Fonts — внешний ресурс (нарушает правила проекта). Решить с мастером до вёрстки финальной версии.

### Типографические тренды 2025

- Крупные заголовки: `clamp(3rem, 8vw, 7rem)` — норма, не эксперимент
- Смешение serif-заголовков и sans-текста (только sans = устаревшее в beauty)
- Один акцентный курсив в заголовке: `ты <em>полюбишь</em>`
- Большой межстрочный интервал: `line-height: 1.6–1.8` — ощущение «воздуха»
- Uppercase лейблы: `text-transform: uppercase; letter-spacing: 0.12em` — журнальный стиль

## 11. ЭКСПЕРТНАЯ ОЦЕНКА

*Оценка текущего лендинга Ольги с трёх независимых точек зрения.*

---

### 🎨 Эксперт 1: UX-дизайнер (10 лет опыта)

#### Типичные ошибки новичков в структуре

**1. Лендинг без лица.** Самая частая ошибка — страница есть, а мастера нет. Клиент не понимает, к кому записывается. Нет фото, нет имени с историей, нет голоса. **Статус у нас: имя есть, фото — нет. Критически нужно исправить.**

**2. Портфолио без фото.** Заглушки с эмодзи работают как временное решение, но мастер маникюра продаёт визуальный результат. Без реальных фото страница не конвертирует — клиент не может оценить уровень.

**3. Нет иерархии действий.** Если на странице 4 кнопки с одинаковым весом — пользователь не знает, куда нажать. Должна быть одна главная CTA и вспомогательные менее заметные.

**4. Секции не рассказывают историю.** Хороший лендинг ведёт пользователя: зацепил → показал качество → объяснил кто ты → доказал что можно доверять → объяснил как записаться → дал возможность записаться. Наш лендинг нарушает порядок: «О мастере» должна идти раньше «Как записаться».

**5. Нет мобильной sticky-кнопки.** Пользователь читает страницу на телефоне, хочет записаться — но кнопка «Записаться» давно ушла вверх. Sticky-кнопка внизу экрана — обязательный элемент.

#### Что ОБЯЗАТЕЛЬНО должно быть на лендинге

| Элемент | Почему |
|---|---|
| Фото мастера | Доверие. Люди идут к человеку, не к странице |
| Реальные фото работ (от 9 шт.) | Главный продающий аргумент в nail-нише |
| Цены (хотя бы «от N ₽») | Скрытая цена = барьер и потеря клиента |
| Отзывы с именами и датами | Независимое подтверждение качества |
| Sticky CTA на мобильном | 70–85% трафика с телефона, кнопка всегда доступна |
| Блок «О мастере» | Персонализация и доверие к конкретному человеку |
| Микротекст под кнопкой | Снижает страх: «без предоплаты», «отвечаю за час» |

#### Что раздражает пользователей и заставляет уйти

- **Медленная загрузка** — несжатые фото убивают конверсию
- **Слишком мелкий текст** — на мобильном шрифт меньше 16px нечитаем
- **Нет ответа на вопрос «где»** — клиент не понимает, далеко ли ехать
- **Только один способ связи** — если Telegram не установлен, клиент потерян
- **Пустые заглушки** — «[ЦЕНА]» и «[ВРЕМЯ]» выглядят как незаконченная работа

### 📊 Эксперт 2: Маркетолог, специалист по конверсии

#### Какой оффер зацепит клиента

Слабый оффер: *«Маникюр в Ростове-на-Дону»* — это есть у всех.

Средний: *«Маникюр, который держится 3 недели»* — конкретика, но без личности.

**Сильный оффер: «Ольга — мастер маникюра в Ростове-на-Дону. 7 лет, 5000+ клиентов, запись без очереди»** — цифры + имя + удобство + конкретный город.

Ещё сильнее — добавить выгоду для клиента: *«Запишитесь сегодня — придёте с результатом на 3 недели».*

**Рекомендация:** Переработать подзаголовок hero, добавив конкретную выгоду, а не просто описание.

#### Какие триггеры доверия нужны

| Триггер | Реализация | Есть? |
|---|---|---|
| Цифры и факты | «5000+ клиентов», «7 лет опыта», «★5.0» | ✅ В hero |
| Социальное доказательство | Отзывы с именем, фото, датой | ❌ Нет |
| Авторитет | Сертификаты, обучение, бренды материалов | ❌ Нет |
| Срочность/дефицит | «Ближайшие свободные даты...» | ❌ Нет |
| Гарантия | «Если не понравится — переделаю» / политика отмены | ❌ Нет |
| Прозрачность | Цены открыто, адрес понятен | ⚠️ Цены-заглушки |
| Простота действия | «Напиши — отвечу за час» | ✅ Есть в шаге 2 |

#### Путь посетителя от «зашёл» до «оставил заявку»

```
Зашёл со смартфона по ссылке из Instagram
         ↓
[Hero] Видит имя, город, «Запись открыта» — остался (3–5 сек)
         ↓
[Портфолио] Листает фото работ — решает нравится ли стиль
         ↓
[О мастере] Понимает, кто этот человек — формируется симпатия
         ↓
[Цены] Проверяет, вписывается ли в бюджет
         ↓
[Отзывы] Читает, что пишут другие — финальное доверие
         ↓
[Как записаться] Видит, что всё просто — 3 шага
         ↓
[Sticky кнопка] Нажимает «Записаться» — переходит в Telegram
```

**Узкое место в текущей версии:** переход от «посмотрел работы» к «нажал кнопку» — нет отзывов и блока «О мастере», которые закрывают доверие. Пользователь уходит, не дойдя до действия.

#### Конкретные рекомендации маркетолога

1. **Добавить «крюк» в hero:** не просто «Записывайтесь», а «Свободные окошки на следующей неделе — запишитесь сейчас»
2. **Отзывы — сразу после портфолио**, пока визуальное впечатление ещё свежее
3. **Один конкретный оффер для первого визита:** «Первый маникюр — консультация бесплатно» или «Скидка 15% на первое посещение»
4. **Микротекст под кнопкой:** «Отвечаю каждый день с 9:00 до 21:00 · Запись без предоплаты»

## 12. TELEGRAM MINI APP

### Что это такое

Telegram Mini App (TMA) — это веб-приложение (HTML + CSS + JS), встроенное прямо в Telegram. Клиент открывает его, не выходя из мессенджера: нажимает кнопку в боте или переходит по ссылке вида `t.me/botname/appname`. Выглядит как нативный экран Telegram — без адресной строки браузера, с родной кнопкой «Назад».

С марта 2024 года TMA доступны и без бота — по прямой ссылке из любого места (профиль, сообщение, кнопка в канале).

---

### Почему это важно для проекта Ольги

Аудитория мастера — женщины 20–45 лет. Telegram установлен у 80%+ из них. Они уже привыкли общаться с мастером в Telegram. TMA убирает лишний шаг: клиент не переходит на внешний сайт, не открывает браузер — он записывается прямо там, где уже общается.

| Что улучшается | Обычный лендинг | Telegram Mini App |
|---|---|---|
| Трение при записи | Открыть ссылку → браузер → форма | Нажать кнопку в Telegram |
| Данные клиента | Вводить имя и телефон | Имя и username подтянутся автоматически |
| Уведомления | Писать отдельно | Бот пишет прямо в тот же чат |
| Напоминания | Отдельная рассылка | Бот напомнит за 24 часа в том же диалоге |
| Оплата | Переход на внешний эквайринг | Telegram Pay (встроенный) |

---

### Технические возможности TMA

**Telegram WebApp API** даёт доступ к:

- `initDataUnsafe.user` — имя, фамилия, username клиента без регистрации
- `MainButton` — большая кнопка внизу экрана (как «Записаться»), меняет цвет и текст
- `BackButton` — кнопка «Назад» в шапке
- `HapticFeedback` — тактильный отклик при нажатиях (только мобильный)
- `showPopup / showAlert / showConfirm` — нативные диалоги в стиле Telegram
- `sendData` — передать данные из Mini App в бота одним вызовом
- `openLink / openTelegramLink` — переходы без выхода из Telegram
- Тема оформления — автоматически подстраивается под светлую/тёмную тему пользователя

**Ограничения:**
- Нужен Telegram-бот как «хост» приложения (создаётся через @BotFather бесплатно)
- Хостинг — любой статик (GitHub Pages, Vercel, Netlify — то, что уже используется)
- HTTPS обязателен
- Не работает в десктопной версии Telegram на Linux (редкий случай)

---

### Варианты реализации

#### Вариант Г — Telegram Mini App (рекомендуется как следующий шаг)

Лендинг превращается в полноценное мини-приложение внутри Telegram. Клиент нажимает кнопку в боте → видит страницу с услугами и расписанием → выбирает слот → подтверждает → бот пишет ему подтверждение.

| **Плюсы** | Нет трения при переходе, данные клиента уже есть, уведомления в том же чате, можно принимать оплату через Telegram Pay |

**Что делает:** Официальный демо-проект команды Telegram. Симулирует заказ бургеров — показывает базовые паттерны TMA: каталог с карточками, выбор позиций, MainButton для оформления заказа. Доступен любому разработчику для изучения.

**Сильные стороны:**

- Корзина с накоплением — хороший паттерн для каталога услуг

- Нет персонализации под бренд

**Что мы можем сделать лучше:**
- Применить тот же паттерн выбора (карточка → добавить) для услуг маникюра
- Добавить фото работ и информацию о мастере — то, чего нет в демо
- Реальная запись вместо симуляции заказа

- Уникальный дизайн в фирменных цветах мастера

- Telegram-уведомления бесплатны — в отличие от SMS

**Что мы можем сделать лучше:**

- Персональный бренд мастера виден сразу, а не размыт в общем каталоге DIKIDI
- Простота — три шага вместо полноценной CRM

## 14. ЭКРАНЫ И ОБЯЗАТЕЛЬНЫЕ ЭЛЕМЕНТЫ TMA ДЛЯ КАТАЛОГА УСЛУГ

*На основе анализа официальной документации Telegram WebApps, UX-паттернов реальных приложений и рекомендаций HIG (Human Interface Guidelines).*

---

### Типичная структура экранов

```
Главный экран
    └─> Каталог услуг (с фильтрами по категориям)
            └─> Карточка услуги  [MainButton: «Записаться»]
                    └─> Выбор мастера (опционально)
                            └─> Выбор даты → Выбор времени  [MainButton: «Продолжить»]
                                    └─> Подтверждение  [MainButton: «Подтвердить запись»]
                                            └─> Успех  [MainButton: «Готово» → close()]

BackButton активна на каждом экране кроме главного.
```

---

### Экран 1 — Главный

| Элемент | Обязательно | Примечание |
|---|---|---|
| Приветствие с именем клиента | Да | Берётся из `initDataUnsafe.user.first_name` |
| Фото мастера или лого | Да | Создаёт личный контакт |
| Краткое описание (1–2 строки) | Да | Кто, где, специализация |
| Кнопка «Записаться» или категории | Да | Главное действие — заметно |
| Ближайшая запись клиента | Если есть | Retention-элемент |

---

### Экран 2 — Каталог услуг

| Элемент | Обязательно | Примечание |
|---|---|---|
| Горизонтальные фильтры-категории | Да | Чипы: «Все / Маникюр / Педикюр / Дизайн» |
| Название услуги | Да | — |
| Цена (крупно) | Да | Отсутствие цены = потеря клиента |
| Длительность | Да | «60 мин» с иконкой часов |
| Краткое описание (1 строка) | Желательно | Что входит в услугу |
| Кнопка выбора или тап по карточке | Да | — |

**Паттерн:** вертикальный список карточек лучше сетки — удобнее читать цены.

---

### Экран 3 — Карточка услуги

| Элемент | Обязательно | Примечание |
|---|---|---|
| Фото работы (пример результата) | Да | Главный продающий элемент |
| Название + полное описание | Да | Что входит, что используется |
| Цена (очень крупно) | Да | Акцентный цвет |
| Длительность | Да | — |
| MainButton «Записаться» | Да | Нативная кнопка Telegram |
| Имя мастера / мини-профиль | Желательно | — |

---

### Экран 4 — Выбор даты и времени

| Элемент | Обязательно | Примечание |
|---|---|---|
| Горизонтальный скролл дат | Да | Текущая + следующая неделя, тап для выбора дня |
| Сетка временных слотов | Да | Кнопки-плитки (10:00 / 10:30 / 11:00) |
| Недоступные слоты (серые) | Да | Визуально заблокированы |
| MainButton «Продолжить» | Да | Активируется только после выбора слота |

**Важно:** не использовать нативный `<input type="date">` — он не вписывается в стиль Telegram.

### Экран 5 — Подтверждение

| Элемент | Обязательно | Примечание |
|---|---|---|
| Сводка: услуга, дата, время | Да | Всё на одном экране |
| Имя клиента (из Telegram) | Да | Подставляется автоматически |
| Итоговая цена | Да | — |
| Политика отмены (1 строка) | Желательно | «Отмена за 2 часа» |
| MainButton «Подтвердить запись» | Да | Главное действие |
| Кнопки «Изменить дату / услугу» | Желательно | Снижает тревогу перед финальным шагом |

### Экран 6 — Успех

| Элемент | Обязательно | Примечание |
|---|---|---|
| Анимация (чекмарк, конфетти) | Да | Позитивное завершение, а не просто текст |
| Дата и время записи (крупно) | Да | — |
| «Бот напомнит за 24 часа» | Да | Снимает тревогу «а вдруг забуду» |
| MainButton «Готово» | Да | Закрывает приложение через `tg.close()` |

### Что обязательно во всех экранах

| Элемент | Где |
|---|---|
| BackButton (нативная Telegram) | Все экраны кроме главного |
| Адаптация под тему Telegram | Везде (светлая / тёмная) |
| Минимальный размер зоны касания | 44×44px на всех кнопках |
| Шрифт системный | Нативный вид внутри Telegram |
| Отсутствие горизонтального скролла страницы | Только вертикальный |

### Типичные UX-ошибки в сервисных TMA

| Ошибка | Последствие | Решение |
|---|---|---|
| Нет цены в списке каталога | Клиент открывает каждую карточку вручную | Цена обязательна в строке каталога |
| Полноэкранный календарь | Занимает весь экран, неудобно | Горизонтальная полоса дат |
| Белый фон при тёмной теме Telegram | Выглядит как баг | `var(--tg-theme-bg-color)` везде |
| MainButton дублируется кнопкой в контенте | Путаница, кажется что нужно нажать дважды | Одна главная CTA |
| Нет BackButton | Клиент застревает, закрывает приложение | BackButton на каждом вложенном экране |
| Запрос лишних данных (email, телефон) | Барьер — данные уже есть из Telegram | Только `initDataUnsafe.user` |
| Нет empty state | Пустой экран без объяснений | Иллюстрация + текст + CTA |
| Нет анимации успеха | Клиент не понимает, прошла ли запись | Чекмарк или конфетти после подтверждения |

## 15. ЭКСПЕРТНАЯ ОЦЕНКА: TMA КАТАЛОГ УСЛУГ МАСТЕРА МАНИКЮРА

*Идея: клиент открывает Telegram-бота → нажимает кнопку → видит услуги с ценами и кнопкой «Записаться». Оценка с трёх точек зрения.*

---

### 🎨 Эксперт 1: UX-дизайнер

#### Типичные ошибки новичков при разработке TMA-каталога

**1. Первый экран — список услуг без контекста.**
Самая частая ошибка — открываешь TMA, а там сразу стена из 10 пунктов меню. Клиент не понимает: это вообще кто? Куда я попал? Нужен маленький «шапочный» блок: фото мастера, имя, специализация, город. 3–4 секунды — и клиент понял, к кому пришёл.

**2. Цена — мелко, в углу, серым цветом.**
Цена — это главное, что ищет клиент в каталоге. Если она плохо видна — клиент раздражён и уходит. Цена должна быть крупной, контрастной, рядом с названием, а не спрятана под «Подробнее».

**3. Кнопка «Записаться» — одна, в самом низу.**
Новички делают один MainButton на всё приложение. Правильно: кнопка должна быть *контекстной* — на экране каталога она неактивна или скрыта, на карточке конкретной услуги — активируется. Клиент должен понимать: нажав, он записывается именно *на эту* услугу.

**4. Нет состояния «услуга выбрана».**
Клиент нажал на карточку — и ничего не изменилось визуально. Непонятно, выбрана услуга или нет. Обязательно: выделение рамкой, галочкой, изменением фона карточки. И текст MainButton должен меняться: «Записаться» → «Записаться на маникюр».

**5. Игнорирование тёмной темы.**
Если захардкодить белый фон (`background: #fff`), у пользователей с тёмной темой Telegram TMA выглядит как ошибка. Использовать только CSS-переменные Telegram: `var(--tg-theme-bg-color)`.

**6. Нет BackButton на вложенных экранах.**
Клиент открыл карточку услуги — и не может вернуться назад. Закрывает приложение. Потеря записи. На каждом экране глубже первого — нативная BackButton Telegram (`tg.BackButton.show()`).

#### Что обязательно должно быть

| Элемент | Почему |
|---|---|
| Мини-профиль мастера на главном | Клиент должен понять, к кому пришёл, за 3 секунды |
| Цена в каждой строке каталога | Не заставлять открывать каждую карточку ради цены |
| Фото работ на карточке услуги | В beauty-нише визуал убеждает лучше любого текста |
| Активная BackButton везде | Без неё клиент застревает и закрывает TMA |
| Длительность услуги | «60 мин» рядом с ценой снимает вопрос «сколько времени займёт» |
| Анимация выбора на карточке | Клиент понимает: я выбрал именно это |
| Экран успеха с подтверждением | Завершение должно быть видимым и радостным, а не «тишина» |

#### Что бесит пользователей

- Нет ответа на вопрос «где находится мастер» — не хочу ехать непонятно куда
- Нажал «Записаться» — ничего не произошло, непонятно, прошла ли запись
- Белый экран при тёмном Telegram — «приложение сломалось»
- Слишком много экранов до записи (> 4 шагов) — бросают на середине
- Нет цен в списке — «значит, дорого и стесняются говорить»
- Приложение не реагирует на нажатие — нет тактильного отклика и визуального фидбека
- Клавиатура перекрывает поле ввода — текст уходит за экран

---

### 📊 Эксперт 2: Маркетолог (конверсия)

#### Что заставит клиента нажать «Записаться»

**Правило 1: Персонализация на входе.**
Telegram автоматически отдаёт имя клиента. Используй это: «Привет, Анна! Выбери услугу 👇» — вместо безликого «Добро пожаловать». Персонализация увеличивает вовлечённость с первой секунды.

**Правило 2: Фото работ — главный триггер.**
Клиент выбирает мастера маникюра *глазами*. Карточка услуги без фото — это карточка без аргумента. Одно красивое фото результата конвертирует лучше трёх абзацев описания.

**Правило 3: Цена + длительность = снятие двух главных вопросов.**
«Сколько стоит?» и «Сколько времени займёт?» — это два главных вопроса клиента. Если оба ответа видны сразу в каталоге, барьер к записи падает вдвое.

**Правило 4: Социальное доказательство на карточке.**
Под описанием услуги — одна строка: «⭐ 4.9 · 87 клиентов» или «Хит — берут чаще всего». Это не хвастовство, это сигнал безопасности: другие уже выбрали, значит всё ок.

**Правило 5: Микротекст под MainButton.**
Под кнопкой «Записаться» — одна строка серым: *«Без предоплаты · Отмена за 2 часа»*. Убирает последний страх перед нажатием.

**Правило 6: Срочность без давления.**
«Ближайшее свободное время: завтра в 14:00» — это не манипуляция, это полезная информация. Клиент видит, что записаться реально, и не откладывает.

#### Элементы, повышающие конверсию

| Элемент | Эффект | Сложность |
|---|---|---|
| Имя клиента в приветствии | +персонализация | Низкая (из `initDataUnsafe`) |
| Фото результата на карточке | +визуальное доверие | Средняя (нужны фото) |
| Цена + длительность в каталоге | -барьер к записи | Низкая |
| «Хит» / «Новинка» лейбл на карточке | +интерес | Низкая |
| Рейтинг / число клиентов | +социальное доказательство | Низкая |
| Микротекст «без предоплаты» под кнопкой | -тревога | Низкая |
| Ближайший свободный слот на главном | +срочность без давления | Средняя |
| Анимация успеха после записи | +позитивное завершение, хочется вернуться | Низкая |

#### Путь клиента от «открыл бота» до «записался»

```
Открыл бота → нажал кнопку меню
       ↓
[Главный] Видит фото мастера, имя, приветствие со своим именем
       ↓
[Каталог] Видит услуги с ценами и длительностью → выбирает
       ↓
[Карточка] Видит фото работы, подтверждает выбор → нажимает MainButton
       ↓
[Дата/время] Выбирает удобный слот
       ↓
[Подтверждение] Видит сводку, нажимает «Подтвердить»
       ↓
[Успех] Получает подтверждение + «бот напомнит за день»
```

**Узкое место:** переход «карточка → дата» — здесь больше всего уходов. Решение: горизонтальная полоса с ближайшими датами прямо под описанием услуги, не на отдельном экране.

---

### 👨‍💻 Эксперт 3: Telegram-разработчик

#### Ограничения платформы — знать заранее

**1. `sendData` работает только из KeyboardButton.**
Если Mini App открыт через inline-кнопку в сообщении (`InlineKeyboardButton`), `tg.sendData()` не работает. Для передачи данных нужен либо собственный backend (fetch на сервер), либо открывать TMA только через Reply Keyboard. Для простого каталога без сервера — открывать через обычную кнопку меню бота.

**2. Нет встроенного хранилища данных.**
TMA — это HTML-страница. Список услуг, расписание, занятые слоты — всё это должно где-то лежать. Варианты без бэкенда: Google Sheets через API (только чтение — можно, запись — сложнее), Telegram CloudStorage (`tg.CloudStorage`) — простое key-value хранилище на стороне Telegram, до 1024 записей.

**3. Размер и скорость загрузки.**
TMA открывается в WebView внутри Telegram. Каждый лишний мегабайт — это секунды ожидания. Правила: картинки только WebP и оптимизированные, никаких тяжёлых шрифтов с Google Fonts, весь JS — встроен или минимален.

**4. Нельзя сохранять состояние между сессиями в localStorage на некоторых платформах.**
`localStorage` работает не везде одинаково внутри Telegram WebView (особенно iOS). Для сохранения выбора между сессиями — использовать `tg.CloudStorage` или передавать данные боту.

**5. Тема меняется — нужно слушать событие.**
Пользователь может переключить тему Telegram прямо во время работы с TMA. Обязательно: `tg.onEvent('themeChanged', applyTheme)` — иначе цвета «застынут» на старой теме.

#### Что работает хорошо

| Возможность | Как использовать |
|---|---|
| `initDataUnsafe.user` | Имя клиента без регистрации — подставить в приветствие |
| `MainButton` | Фиксированная кнопка Telegram — главное действие на каждом экране |
| `BackButton` | Нативная кнопка «Назад» — обязательна на вложенных экранах |
| `HapticFeedback` | Тактильный отклик при выборе услуги — `impactOccurred('light')` |
| `tg.showAlert()` | Нативный диалог подтверждения — «Вы уверены, что хотите отменить?» |
| CSS-переменные темы | Автоматическая адаптация к светлой/тёмной теме без JS |
| `tg.expand()` | Разворачивает TMA на весь экран при открытии — обязательный вызов |
| `tg.enableClosingConfirmation()` | Запрашивает подтверждение при случайном закрытии во время записи |

#### Что лучше не делать

| Антипаттерн | Почему | Как правильно |
|---|---|---|
| Использовать `<input type="date">` | Нативный пикер не вписывается в стиль Telegram | Кастомная горизонтальная полоса дат |
| Хардкодить цвета (`#fff`, `#000`) | Ломается при тёмной теме | `var(--tg-theme-bg-color)` и аналоги |
| Делать кнопки меньше 44px | Неудобно нажимать на мобильном | Минимум 44×44px для всех интерактивных элементов |
| Загружать все данные при старте | Долгий старт, пустой экран | Загружать по мере перехода между экранами |
| Дублировать MainButton кнопкой в DOM | Путаница, нажимают не ту | Только один способ подтвердить действие |
| Открывать внешние ссылки без `tg.openLink()` | Пользователь уходит из Telegram | Использовать `tg.openLink()` или `tg.openTelegramLink()` |
| Не вызывать `tg.ready()` сразу | Загрузочный экран Telegram висит дольше | `tg.ready()` — первая строка скрипта |

#### Минимальный стек для MVP каталога (без бэкенда)

```
HTML + CSS + vanilla JS
│
├── telegram-web-app.js  (официальный SDK, единственная зависимость)
├── Данные услуг         (JSON-объект прямо в скрипте)
├── Расписание           (Google Sheets API, только чтение)
└── Запись клиента       (tg.sendData → бот получает и пишет Ольге)
```
